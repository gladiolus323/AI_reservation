**작성일**: 2025-12-09
**프로젝트명**: AI 예약 에이전트 (AI Reservation Agent)

---

## 1. 프로젝트 개요

### 1.1 목적
병원 검사 예약 과정을 AI 에이전트가 자동화하여 환자 편의성 향상 및 예약 업무 효율화

### 1.2 주요 기능
- 환자 정보 조회 및 기존 예약 확인
- 처방코드 기반 검사 정보 조회
- 검사 간 제약조건(태그/규칙) 자동 검증
- 최적 예약 시간대 3개 추천
- 버튼 기반 UI로 간편한 예약 선택

## 3. 데이터 구조

### 3.1 데이터 파이프라인
```
source.xlsx (원본)
     │
     ├──► patient_schedule.csv (가공) - 환자 예약 정보
     ├──► resource_schedule.csv (합성) - 장비 배정 정보
     └──► constraints.csv (가공+합성) - 검사 제약조건

hospital_rules.txt (설정) - 병원별 커스텀 정책
```

### 3.2 주요 데이터 파일

| 파일 | 유형 | 레코드 수 | 설명 |

| patient_schedule.csv | 가공 | ~7,900건 | 환자별 예약 정보 |
| resource_schedule.csv | 합성 | ~7,900건 | 장비별 예약 현황 |
| constraints.csv | 가공+합성 | 250개 | 검사 제약조건 |
| hospital_rules.txt | 설정 | - | 예약 정책 |

### 3.3 장비 현황
- **총 장비유형**: 19개
- **총 리소스**: 40개
- **주요 장비**: CT(5), MRI(4), 청력검사실(4), 초음파(3)

## 4. MCP 도구 (Tools)

AI 에이전트가 사용하는 4개의 MCP 도구:

| 도구명 | 기능 | 사용 데이터 |
|--------|------|-----------|
| get_patient_info | 환자 정보 및 기존 예약 조회 | patient_schedule.csv |
| get_exam_by_code | 처방코드로 검사 정보 조회 | constraints.csv |
| get_exam_constraints | 검사 제약조건 조회 | constraints.csv |
| get_unavailable_slots | 예약 불가 시간대 조회 | 전체 CSV |

## 5. 주요 개선 사항

### 5.1 버튼 기반 UI (2025-12-03)
**문제점**: 환자가 "추천 1번으로 예약할게요" 직접 타이핑 필요

**해결**:
- 추천 응답 시 4개 버튼 자동 표시
- [추천1] [추천2] [추천3] [재추천]
- 클릭 한 번으로 선택 완료

### 5.2 재추천 횟수 제한 (2025-12-03)
**문제점**: 무한 재추천 요청 가능

**해결**:
- 최대 3회 재추천 제한
- 초과 시 전화 상담 안내
- 추천 선택 시 카운터 리셋

### 5.3 예약 불가 시 자동 대안 제시 (2025-12-03)
**문제점**: 불가능 안내 후 환자에게 재질문

**해결**:
- "요청하신 날짜에 예약이 불가능해요" 먼저 안내
- 즉시 대안 3개 자동 제시
- 재질문 절대 금지 규칙 추가

### 5.4 금식 검사 규칙 강화 (2025-12-03)
**문제점**: 금식 검사에 오후 시간 추천

**해결**:
- 금식 검사는 오전 10시 이전만 추천
- 환자 선호보다 의료 규칙 우선
- hospital_rules.txt에 특별 규칙 추가

### 5.5 데이터 스키마 문서화 (2025-12-09)
**작업 내용**:
- 전체 데이터 파이프라인 구조 문서화
- 5개 데이터 파일 스키마 상세 정의
- MCP 도구와 데이터 매핑 관계 정리
- 파일 유형 분류 (원본/가공/합성/설정)

## 6. 예약 플로우

```
1. 환자번호 입력
   └─► get_patient_info 호출 → 기존 예약/외래일 확인

2. 처방코드 입력
   └─► get_exam_by_code 호출 → "OOO 검사가 맞으신가요?"

3. 검사 확정 (버튼 클릭)
   └─► get_exam_constraints 호출 → "선호하시는 날짜가 있으신가요?"

4. 선호 일정 입력
   └─► get_unavailable_slots 호출 → 예약 가능 시간 계산

5. 추천 3개 제시
   └─► 버튼으로 선택 → 예약 확정
```

### 7.1 태그 기반 규칙

### 7.2 검사 간격 규칙
| 조건 | 간격 | 사유 |
|------|------|------|
| Nuclear → Nuclear | 1일 | 방사성 동위원소 간섭 방지 |
| Barium → Abdomen | 3일 | 바륨 잔류로 영상 품질 저하 |
| Sedation → Treadmill | 1일 | 진정 후 협조 곤란 |
