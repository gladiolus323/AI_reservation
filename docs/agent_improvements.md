## 1. 버튼 기반 추천 선택 UI

### 문제점
- 기존에는 에이전트가 텍스트로 "[추천1] [추천2] [추천3] [재추천]"을 출력
- 환자가 직접 "추천 1번으로 예약할게요"라고 타이핑해야 함
- 사용자 경험(UX)이 불편하고 입력 오류 가능성 존재

### 개선 내용

#### 1.1 시스템 프롬프트 수정 (`backend/src/prompts/system-prompt.js`)
- 추천 응답 형식을 표준화하여 **추천 1**, **추천 2**, **추천 3** 형식으로 통일
- 버튼 선택 처리 로직 추가 (환자가 "1번", "추천1", "첫번째" 등 다양한 표현 인식)

#### 1.2 UI 버튼 구현 (`web/index.html`)

**CSS 스타일 추가:**

**JavaScript 로직 추가:**
- 에이전트 응답에서 `**추천 1**`, `**추천 2**`, `**추천 3**` 패턴 감지
- 패턴 감지 시 4개의 버튼 자동 표시:
  - [1️⃣ 추천 1 선택] - 초록색
  - [2️⃣ 추천 2 선택] - 파란색
  - [3️⃣ 추천 3 선택] - 보라색
  - [🔄 재추천] - 흰색 테두리

**버튼 클릭 시 동작:**
```javascript

    // 선택에 따른 메시지 전송
    // - 1, 2, 3: "추천 N번으로 예약할게요"
    // - 0 (재추천): "다른 시간대로 재추천해주세요"
}
```

## 2. 재추천 횟수 제한

### 문제점
- 환자가 무한히 재추천을 요청할 수 있음
- 실제로 원하는 시간대를 찾기 어려운 경우 대안 필요

### 개선 내용

#### 2.1 재추천 카운터 구현
```javascript
let retryCount = 0;
const MAX_RETRY = 3; // 최대 재추천 횟수
```

#### 2.2 제한 도달 시 전화번호 안내
재추천 3회 초과 시 다음 메시지 표시:

> 원하시는 시간대를 찾기 어려우시네요. 😔
>
> 검사 예약 관련하여 직접 상담을 원하시면 아래 번호로 연락해주세요.
>
> 📞 **검사예약 문의**: xxx-xxxx
> ⏰ **운영시간**: 평일 09:00 ~ 17:00
>
> 전문 상담원이 최적의 시간을 찾아드리겠습니다.

#### 2.3 카운터 리셋 조건
- 환자가 추천 1, 2, 3 중 하나를 선택한 경우
- 새 대화 시작 시 (resetChat 함수)

## 3. 예약 불가 시 자동 대안 제시

### 문제점
- 테스트 결과 4건(TC008, TC014, TC018, TC023)에서 "추천 슬롯 없음" 발생
- 에이전트가 불가능한 이유는 설명하지만, 환자에게 다시 날짜를 물어보는 문제
- 예: "1월 8일 이후 중 선호하시는 날짜가 있으신가요?" → 환자 재질문 유발

### 개선 내용 (`backend/src/prompts/system-prompt.js`)

시스템 프롬프트에 "예약 불가 시 자동 대안 제시" 섹션 추가:

```
## 예약 불가 시 자동 대안 제시 (필수)

**중요**: 환자가 원하는 날짜/시간에 예약이 불가능한 경우, 절대로 환자에게 다시 날짜를 물어보지 마세요.
반드시 다음 절차를 따르세요:

1. **먼저 불가 안내**: "환자분이 요청하신 일자에는 예약이 불가능해요. 대신 이런 날짜는 어떠세요?"
2. **자동으로 대안 3개 제시**: 병원 규칙을 준수하는 가능한 날짜/시간 3개를 즉시 추천
3. **추천 형식 유지**: 반드시 **추천 1**, **추천 2**, **추천 3** 형식으로 제시

**절대 금지**: "다른 날짜를 말씀해주세요", "언제가 좋으실까요?" 등 환자에게 재질문하는 것
```

### 기대 효과
- 환자가 다시 날짜를 입력할 필요 없이 바로 대안 선택 가능
- 대화 턴 수 감소로 예약 완료까지 시간 단축
- 버튼 UI와 연동되어 일관된 UX 제공

## 4. 금식 검사 규칙 강화

### 문제점
- 테스트 결과 금식 검사에서 오후 시간을 추천하는 경우 발생 (TC005)
- 환자가 "오후 선호"라고 해도 의료 규칙이 우선되어야 함

### 개선 내용 (`data/hospital_rules.txt`)

```
[우선순위]
1. 금식 검사는 무조건 오전 10시 이전에만 추천해야 합니다 (환자 선호보다 우선)
2. 검사 간 의료적 간격 규칙 준수가 중요합니다
3. 외래 진료 전 검사 완료를 고려하세요
4. 환자의 선호 시간대는 의료 규칙을 만족하는 범위 내에서만 반영해주세요

[금식 검사 특별 규칙]
- 금식이 필요한 검사: 환자가 오후를 원해도 반드시 오전 10시 이전으로 추천
- 금식 검사에서 환자가 "오후"를 선호한다고 해도, 의료적 이유로 오전만 가능함을 안내
- 예시 응답: "금식이 필요한 검사이므로 오전 10시 이전에만 예약이 가능합니다."
```
