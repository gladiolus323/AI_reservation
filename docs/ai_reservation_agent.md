# AI 예약 추천 에이전트

## 개요

| 항목 | 내용 |
|------|------|
| **목적** | 병원 검사 예약 시 AI가 최적 시간대 추천 |
| **사용자** | 환자 본인 |
| **인터페이스** | 웹 채팅 |
| **핵심 기능** | 빈 시간 + 사용자 니즈 + 병원 규칙 반영한 추천 |

## 오류 처리

| 상황 | 대응 |
|------|------|
| 환자번호 없음 | "찾을 수 없습니다. 다시 입력해주세요" |
| 처방코드 없음 | "처방코드를 찾을 수 없습니다. 다시 확인해주세요" |
| 선호 조건 불충족 | 대안 추천 ("오전은 마감되어 오후로 추천드려요") |
| 빈 자리 없음 | "추천 기간 내 예약 가능한 자리가 없습니다" |

## 병원 규칙 커스터마이징

### 파일 위치
`data/hospital_rules.txt`

### 편집 방법
메모장으로 직접 편집

### 적용 방식
수정 시 프롬프트에 자동 반영

### 예시
```
[병원 설정]
추천 기간: 30일 이내

[추천 규칙]
검사는 외래 진료일 이전에 완료될 수 있도록 추천해주세요
금식이 필요한 검사는 오전 10시 이전으로 추천해주세요
같은 날 검사는 최대 2개까지만 추천해주세요

[우선순위]
1. 검사 간 의료적 간격 규칙 준수가 가장 중요합니다
2. 외래 진료 전 검사 완료를 두 번째로 중요하게 고려하세요
3. 환자의 선호 시간대는 가능한 범위에서 반영해주세요
```

## MCP 서버 도구 (Tools)

| 도구명 | 기능 | 입력 | 출력 |
|--------|------|------|------|
| `get_patient_info` | 환자 정보 조회 | 환자코드 | 기존 예약, 외래일 |
| `get_exam_by_code` | 처방코드로 검사 조회 | 처방코드 | 검사명, 장비유형 |
| `get_exam_constraints` | 검사 제약조건 조회 | 처방코드 | 장비유형, 소요시간, 태그, 규칙 |
| `get_unavailable_slots` | 예약 불가능 시간 조회 | 장비유형, 기간, 환자코드, 처방코드 | 예약된 시간, 환자 충돌, 간격 규칙 위반 |

> **Note**: 처방코드 입력 방식으로 변경. `search_exam_name` 삭제, `get_exam_by_code` 추가. (2025-12-02)

## 웹 UI 기능

### 빠른 입력 버튼
| 버튼 | 동작 |
|------|------|
| 환자번호 예시 | "환자번호 00003795입니다" 입력 |
| 처방코드 예시 (CT) | "HA443" 입력 |
| 처방코드 예시 (MRI) | "MR220101" 입력 |
| 오전 선호 | "오전이 좋아요" 입력 |

### 처방명 확인 버튼
AI가 "OOO 검사가 맞으신가요?" 질문 시 자동으로 표시:

| 버튼 | 클릭 시 동작 |
|------|-------------|
| ✓ 네, 맞습니다 | "네, 맞습니다" 전송 → 선호 일정 질문 |
| ✗ 아니요, 다시 입력할게요 | "아니요, 처방코드를 다시 입력할게요" 전송 → 재입력 안내 |

## 개발 계획

| 단계 | 작업 | 설명 |
|------|------|------|
| 1 | MCP 서버 구축 | 데이터 로더 + 4개 도구 구현 |
| 2 | 백엔드 서버 구축 | Express + Claude API + MCP 연동 |
| 3 | 웹 채팅 UI 구현 | HTML/CSS/JS 채팅 인터페이스 |
| 4 | 통합 테스트 | 전체 흐름 테스트 |

## 개발 상세 순서

### 1단계: MCP 서버 구축

```
mcp-server/
├── package.json
└── src/
    ├── index.js              # MCP 서버 메인
    ├── data-loader.js        # CSV 파일 로딩
    └── tools/
        ├── get-patient-info.js
        ├── get-exam-by-code.js
        ├── get-exam-constraints.js
        └── get-unavailable-slots.js
```

**순서:**
1. `package.json` 생성 (의존성: `@modelcontextprotocol/sdk`)
2. `data-loader.js` - CSV 파일 3개 로딩 및 파싱
3. 도구 구현 (하나씩):
   - `get-patient-info.js` - 환자 정보 조회
   - `get-exam-by-code.js` - 처방코드로 검사 조회
   - `get-exam-constraints.js` - 검사 제약조건 조회
   - `get-unavailable-slots.js` - 예약 불가능 시간 조회
4. `index.js` - MCP 서버 메인 (도구 등록)
5. MCP 서버 단독 테스트

---

### 2단계: 백엔드 서버 구축

```
backend/
├── package.json
├── server.js                 # Express 웹서버
└── claude-client.js          # Claude API + MCP 연동
```

**순서:**
1. `package.json` 생성 (의존성: `express`, `@anthropic-ai/sdk`, `@modelcontextprotocol/sdk`)
2. `claude-client.js` - Claude API 호출 + MCP 도구 연동
3. `server.js` - Express 서버 (채팅 API 엔드포인트)
4. `hospital_rules.txt` 자동 로딩 → 프롬프트 삽입 로직
5. 백엔드 API 테스트

---

### 3단계: 웹 채팅 UI 구현

```
web/
└── chat.html                 # 채팅 UI
```

**순서:**
1. HTML 구조 (채팅 메시지 영역, 입력창)
2. CSS 스타일링
3. JavaScript (메시지 전송, 응답 표시)
4. 백엔드 API 연동

---

### 4단계: 통합 테스트

1. 전체 흐름 테스트 (환자번호 → 처방코드 → 선호 → 추천)
2. 오류 케이스 테스트 (잘못된 입력 등)
3. 병원 규칙 변경 테스트
